Javascript 事件队列机制（Event-loop）

思考以下代码

Javascript 设计之初是服务于浏览器网页，这就导致了它是单线程的原因（与 DOM 操作有关），单线程的弊端在于，如果一个操作很耗时，那么后续的只能等待，为解决此问题，出现了 JS 异步编程

Javascript 异步编程方式

  • 回调函数
  • Promise
  • sync/await

Javascript 何时执行异步流程，异步流程内部如何处理

js 主线程运行到异步时，会将异步交给对应的模块处理比如遇到 setTimeout，setInterval 时，会交由 timer 模块处理，然后 timer 模块会根据具体的情况，将它们一一推入事件队列，当主线程执行完毕，会主动遍历事件队列，一一取出来严格遵循先进先出的规则逐步执行。
